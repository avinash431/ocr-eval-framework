# Run Evaluation

Execute OCR evaluations at three levels of scale and help interpret results.

## Before You Start

1. Check the virtual environment is active: `source venv/bin/activate`
2. Read `configs/config.yaml` (or `config.local.yaml`) to confirm paths and settings
3. Verify API keys are set in `.env` for cloud models
4. Confirm test documents exist in the dataset directory

## Evaluation Modes

### Single Document Test

Quick test of one model on one document. Use for debugging or spot-checking.

```bash
python run_single.py --model <model_name> --input <path_to_image> [--config configs/config.local.yaml] [--output output.txt]
```

When to use: user says "test X on this image", "try OCR on this document", "quick check"

### Full Model Evaluation

Run one model across the entire dataset with metrics.

```bash
python run_model.py --model <model_name> [--config configs/config.local.yaml]
```

Results saved to: `results/YYYYMMDD_HHMMSS_<model_name>/`

When to use: user says "evaluate tesseract", "run mistral on everything", "full test of X"

### Batch Comparison

Run multiple (or all) models across the dataset for side-by-side comparison.

```bash
# All models
python run_batch.py [--config configs/config.local.yaml]

# Specific models
python run_batch.py --models tesseract mistral_ocr surya

# List available models
python run_batch.py --list
```

Results saved to: `results/YYYYMMDD_HHMMSS_batch/`

When to use: user says "compare all models", "benchmark", "run batch", "which model is best"

## After Running

### Generate Report

```bash
python evaluate.py --results-dir results/<latest_run_dir> [--export-csv]
```

Produces `report.html` with comparison tables, color-coded metrics, and per-category breakdowns.

### Interpret Results

Guide the user through the metrics:

| Metric | Range | Better | What It Measures |
|--------|-------|--------|-----------------|
| CER | 0.0–1.0+ | Lower | Character-level accuracy (Levenshtein) |
| WER | 0.0–1.0+ | Lower | Word-level accuracy |
| F1 | 0.0–1.0 | Higher | Token overlap (precision + recall) |
| BLEU | 0.0–1.0 | Higher | N-gram similarity |
| Edit Distance | 0.0–1.0 | Lower | Normalized string distance |

Quality thresholds:
- **Good**: CER < 0.05, F1 > 0.85
- **Acceptable**: CER < 0.15, F1 > 0.70
- **Poor**: CER > 0.15, F1 < 0.70

### Troubleshooting

- **Model not found**: Check `python run_batch.py --list` — model file may have import errors
- **API key errors**: Verify `.env` has the correct key and `load_config()` picks it up
- **Timeout**: Increase `execution.timeout_seconds` in config
- **No metrics**: Ground truth files must exist in `ground_truth_dir` with matching filenames (`<stem>_gt.txt`)
- **GPU OOM**: Try `device: cpu` in config, or use a smaller model variant

## Result Directory Structure

```
results/YYYYMMDD_HHMMSS_batch/
├── raw_outputs/          # Extracted text per model+document
│   ├── tesseract__doc1.txt
│   └── mistral_ocr__doc1.txt
├── metrics/              # Per-model metric JSONs
│   ├── tesseract_metrics.json
│   └── mistral_ocr_metrics.json
├── tesseract_results.json
├── mistral_ocr_results.json
├── batch_summary.json    # Aggregated comparison
└── report.html           # Generated by evaluate.py
```
